# Store_Info/ — всё для публикации в магазине

В этой папке лежат файлы, **связанные с публикацией приложения в РуСтор** (и потенциально другие сторы): иконки, скриншоты, описание, политика конфиденциальности, ключ подписи APK.

**Никакая** часть этой папки (кроме `icon.png`) не попадает в собранный APK — это **метаданные для магазина** и для тебя как разработчика. Если ты создаёшь новые игровые ассеты (картинки, звуки, музыка) — клади их в другую папку (например, `assets/` в корне проекта).

## Содержимое

| Файл / папка | Размер / тип | Куда уходит |
|--------------|--------------|-------------|
| `keystore/release.jks` | бинарь | **В APK** (как подпись) — html2apk auto-detect при `-Release`. См. `keystore/README.md`. |
| `keystore/keystore.config.json` | alias + пароли | Читается html2apk. |
| `icon.png` | 1024×1024 PNG | **В APK** — html2apk через `iconPath` в `.claude/build-config.json` уменьшает в `mipmap-*` (mdpi…xxxhdpi). |
| `store-icon.png` | 512×512 PNG | **В магазин** — RuStore Console, страница приложения. Загружается вручную. |
| `store-feature.png` | 1024×500 PNG | **В магазин** — баннер на странице приложения (опционально). |
| `screenshots/01..04.png` | 1080×1920 PNG | **В магазин** — скриншоты на странице приложения. Минимум 4, рекомендуется 6-8. |
| `STORE_LISTING.md` | markdown | Тексты для страницы в магазине (краткое/полное описание, теги). Копируется руками в RuStore Console. |
| `PRIVACY_POLICY.md` | markdown | Готовый текст политики конфиденциальности + инструкция как захостить (облако или GitHub Pages). |

## Сейчас здесь заглушки

PNG-файлы — серые/тёмные с надписями размера и назначения. **Замени** на реальные арты до публикации:

1. **`icon.png`** (1024×1024) — критично, попадает в APK. Делает дизайнер или AI-генератор. Должна читаться на маленьких размерах (mdpi 48×48).
2. **`store-icon.png`** (512×512) — обычно та же иконка что и `icon.png`, но в этом разрешении. Можно сделать с более насыщенным цветом — лучше выделяется в магазине.
3. **`store-feature.png`** (1024×500) — баннер с названием игры + ключевой визуал. Опционально для РуСтор, но повышает CTR.
4. **`screenshots/*`** — реальные кадры из игры на устройстве (не эмулятор).

## Что НЕ нужно делать

- ❌ Не клади сюда saves / temp / большие игровые ассеты.
- ❌ Не переименовывай `icon.png`, `release.jks` — html2apk ищет их по этим именам.
- ❌ Не меняй размеры PNG — РуСтор отвергает изображения с нестандартными dimensions.
- ❌ Не удаляй `keystore/release.jks` — потеряешь подпись, и все опубликованные приложения нельзя будет обновлять.

## Как использовать

После замены заглушек на реальные арты:
- При следующей сборке APK (через skill `prepare-release-candidate` или `build-apk-from-html`) — `icon.png` и keystore подхватятся автоматически.
- `store-icon.png` / `store-feature.png` / `screenshots/*` — **руками** загрузи в RuStore Console на странице приложения. Они не попадают в APK.
- Тексты из `STORE_LISTING.md` — скопируй в форму RuStore Console.
- `PRIVACY_POLICY.md` — захости на облаке (см. инструкцию в самом файле) и URL впиши в `STORE_LISTING.md` → privacyUrl.
