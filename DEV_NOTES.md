# DEV_NOTES

Карта проекта для разработчика. Где что лежит и куда тыкаться. Здесь же —
полный свод правил для нового контента карточек.

---

## 1. Архитектура коротко

- `index.html` — все экраны + модалки + кеш-bust query на скриптах (`?v=NN`).
- `styles.css` — единый stylesheet, тёмная неоновая тема.
- `config.js` — глобальные настройки (реклама, режимы, blacklist).
- `cards.js` — массив `window.CARDS` со всем контентом.
- `cardEngine.js` — фильтрация по режиму, выборка без повторов в сессии.
- `storage.js` — обёртка над localStorage + сессионная память для имён.
- `ads.js` — `AdManager` (native bridge Yandex Mobile Ads или mock).
- `audio.js` — короткие WebAudio-«плинки» и вибрация.
- `ui.js` — переключение экранов, конфетти, тосты.
- `game.js` — состояние игры, бутылочка, ход карточек.
- `main.js` — точка входа, навеска обработчиков, dev-панель `?dev=1`.

## 2. Карточки

### Схема

```js
{
  id:   <уникальное число>,
  type: 'truth' | 'dare',
  mode: 'romance' | 'flirt' | 'passion',
  text: 'Текст карточки на русском.'
}
```

Поля `category` и `intensity` **удалены** — каждый режим самодостаточен,
все карточки в нём максимальной интенсивности.

### Где живёт

- `cards.js` — массив `window.CARDS`.
- Геттеры: `window.CardsData.all() / byType(t) / byMode(m) / count()`.

### Где фильтруется

- `cardEngine.js`:
  - `pool({ type, mode })` — карточки нужного типа и режима, плюс
    временно открытые режимы из `tempUnlocks` (rewarded в будущем).
  - `pick(options)` — случайная из пула без повторов в сессии; если
    пул кончился — сбрасывает `sessionSeen` и берёт заново.
  - `markSeen(id)` — пометить, что эту карточку уже показали в сессии.

### Где задаются режимы

- `config.js → modes`:
  ```js
  romance: { label: 'Романтика', emoji: '❤️' },
  flirt:   { label: 'Флирт',     emoji: '🫦' },
  passion: { label: 'Страсть',   emoji: '🔥' }
  ```
- В UI — 3 кнопки `.mode-btn` на главном с эмодзи + подписью + sub.
- Storage хранит выбранный режим; миграция со старых имён `soft`/`mixed`
  /`funny` в `romance` встроена в `Storage.getMode()`.

## 3. Правила контента (ВАЖНО)

Эти правила — **не рекомендации**, а условие приёма карточек. Если карточка
им не удовлетворяет — её нужно переписать, а не сохранять «слабый вариант».

### 3.1. Truth — это вопрос, обращённый ПРЯМО к игроку

- Игрок, на которого выпала карточка, — **тот, кто отвечает**.
- Формулировка во втором лице: `«Ты ...»`, `«Какой твой ...»`, `«Что в тебе ...»`,
  `«Когда ты ...»`, `«О чём ты думаешь ...»`.
- ❌ «Что Алина чувствует, когда ...» — третье лицо.
- ❌ «Спросите друг друга, как ...» — оба игрока.
- ✅ «Что ты чувствуешь, когда мы держимся за руки?»

### 3.2. Dare — это действие, где основной актёр = игрок, кому выпала карточка

- Партнёр — пассивный получатель. Игрок инициирует, делает, говорит.
- ❌ «Обнимитесь молча 2 минуты» — оба активны.
- ❌ «Посмотрите друг другу в глаза 30 секунд» — оба.
- ❌ «Сделайте 5 синхронных вдохов» — оба.
- ❌ «Дай партнёру укусить твою губу» — активен партнёр, не игрок.
- ✅ «Обними партнёра со спины и подержи 20 секунд молча.»
- ✅ «Поцелуй партнёра в шею и медленно проведи языком вниз.»
- ✅ «Укуси партнёра за нижнюю губу при следующем поцелуе.»

Если хочется передать партнёру инициативу — переформулируй так, чтобы
игрок всё равно был источником действия:
- ❌ «Дай партнёру попробовать твою помаду губами»
- ✅ «Поделись с партнёром поцелуем с помадой/блеском.»

### 3.3. Максимальная интенсивность в рамках режима

Никаких «лентяй»-вариантов:
- ❌ «Скажи партнёру что-нибудь приятное» — слабо, общо.
- ✅ «Скажи партнёру три причины, почему ты выбрал(а) бы его(её) снова.»

В каждой карточке должен быть **конкретный поступок** (для dare) или
**конкретный, точечный вопрос** (для truth).

### 3.4. Разделение тонов по режимам

- **Romance (❤️)** — нежное, тёплое, значимое. Глубина и забота, без эротики.
  Темы: воспоминания, благодарность, мечты, забота, ритуалы.
- **Flirt (🫦)** — игривое, заигрывающее, лёгкое 16+. Намёки, флирт, лёгкие
  прикосновения, шёпот, дразнить. Без откровенности.
- **Passion (🔥)** — самое жаркое, 18+, но **без откровенной порнографии**.
  Темы: близость, прикосновения, желание, фантазии. Допускаются: лёгкие
  укусы, расстёгивание/раздевание, ролевые «команды», лёд/шёлк, удержание
  запястий — всё с подразумеваемым согласием.

### 3.5. Запрещено в любом режиме

- Откровенная порнография (анатомические описания акта).
- Принуждение, насилие, унижение.
- Алкоголь, табак, наркотики.
- Задания, требующие денег.
- Задания с риском физической травмы.
- Темы измен, болезней, травм, политики, религии.
- Манипулятивные / токсичные формулировки («Признайся, что ты ...», «Скажи,
  что без меня ты ...»).
- Сравнения с бывшими / другими партнёрами.

### 3.6. Длина и читаемость

- Желательно 1–3 строки в финальной разметке (≈80–150 символов).
- Не использовать сложноподчинённые конструкции из трёх вложений.
- Род-окончания: давать `(а)` для обоих полов: «выбрал(а)», «сделал(а)».

### 3.7. Согласие — встроено

Если карточка предполагает что-то телесно интимное (укус, удержание,
снятие нижнего белья), добавляй маркер согласия в сам текст карточки:
`«(если он(а) хочет)»`, `«(по согласию)»`. Не выносить отдельной строкой.

### 3.8. Проверка перед добавлением

В dev-панели (`?dev=1`) есть кнопка «Проверить контент» — она найдёт:
- дубликаты id;
- пустые тексты;
- слова из `config.js → blacklist`.

Прогоняй её после каждого пакета новых карточек.

## 4. Игровая логика

### Game (`game.js`) — публичный API

- `start()` — построить 12 слотов вокруг бутылочки, показать игровой экран.
- `spinBottle()` — двухстадийный спин (5 оборотов linear + ease-out до случайной точки).
- `completeCurrent()` — «Выполнено». Помечает слот сыгранным, ход переходит.
- `replaceCurrent()` — «Заменить». Можно один раз за карточку.
- `playAgain()` / `backHome()` / `endGame()` — навигация.
- `refresh()` — перечитать имена/пол из Storage и обновить шапку/чип.
- `_state` — для дебага в dev-консоли.

### Слоты вокруг бутылочки

- 12 позиций, ровно 6 truth + 6 dare.
- Раскладка с ограничением: не более 2 одинаковых символов подряд
  (`arrangeSlotTypes()` в `game.js`).
- При попадании бутылки на использованный слот — обход по часовой стрелке.
- Партия — 12 ходов.

### Параметры спина (game.js, top of file)

- `SPIN_FIXED_TURNS = 5` — гарантированные обороты стадии 1.
- `SPIN_OMEGA_DEG_PER_MS = 1.2` — начальная угловая скорость (≈3.33 об/с).
- `DECEL_MIN_MS = 1100` / `DECEL_MAX_MS = 1800` — длительность торможения.
- `DECEL_BEZIER = 'cubic-bezier(0.25, 0.5, 0.5, 1)'` — easeOutQuad,
  скорость на стыке continuous, остановка плавная.

## 5. Storage

- `localStorage` (через `Storage.*`):
  - `mode` — выбранный режим (`romance` по умолчанию).
  - `seenCards` — общая история показанных карточек (не сбрасывается между играми).
  - `sound`, `vibration` — настройки.
  - `onboardingOk` — флаг, что онбординг пройден.
  - `mockAds` — переключатель из dev-панели.
- **Сессионная память** (внутри модуля, сбрасывается на reload страницы):
  - `sessionPlayers` — имена + пол. Этим достигается требование: при
    обновлении страницы — дефолтные «Игрок 1»/«Игрок 2»; в рамках одной
    сессии — последние введённые значения.
- Полный сброс — `Storage.resetAll()` или кнопка в dev-панели.

## 6. Реклама (`ads.js`)

Подробности — в `docs/ADS.md`. Коротко:

- Два backend'а: `native` (Yandex Mobile Ads через html2apk-bridge) и
  `mock` (DOM-оверлей в браузере). `ensureBackend()` определяет автоматически.
- API: `showInterstitialAd()` → Promise, `showRewardedAd(reward)` → Promise.
- Игра дёргает interstitial **между остановкой бутылочки и показом карточки** —
  чтобы не разрывать момент прочтения.
- Правила частоты: `cardsBeforeFirstAd: 5`, диапазон `minCardsBetweenAds`–
  `maxCardsBetweenAds`. Все в `config.js`.

## 7. UI / визуальные правила

- Тёмный фон с радиальными свечениями + парящие сердечки.
- Бренд: розово-фиолетовый (`--neon-pink`, `--neon-purple`) — для бутылочки
  и общего акцента.
- Truth: голубой неон (`#0fc4ff` основной, `#5fdaff` мягкий).
- Dare: розово-фиолетовый (`#d52aff` основной, `#e780ff` мягкий).
- Бейдж над заголовком карточки: круг с `?` (truth) или `!` (dare),
  тип-цветный glow.
- Угловые символы карточки тоже `?` / `!` в цвет типа (Georgia italic 30px).
- Чип «Ход / Имя / ♀/♂» под бейджем — заметный, с border-glow по полу.

### Адаптив

- `(max-width: 350px)` — узкие экраны.
- `(max-height: 720px)` — компакт; ориентируемся на 9:16.
- `(max-height: 580px)` — сверхкомпакт.
- `(min-height: 760px)` — расширение бутылочки.
- Карта с заданием flex-центрирована в `card-stack` между title-block
  и фикс. кнопкой «Выполнено». **Не position:fixed**, иначе на 9:16
  она накрывает чип игрока.

## 9. Известные ограничения

- Карточки грузятся синхронно из `cards.js`. На 500–1000 норм; при росте
  до десятков тысяч — переносить в JSON + ленивая подгрузка.
- `localStorage` ограничен ≈5 МБ — наш объём в десятки раз меньше.
- `requestAnimationFrame` в headless-превью замораживается. Все анимации
  спина бутылки сделаны через CSS-transition + setTimeout — работают в
  обоих окружениях.

---

## 10. Кеш-bust

Каждое изменение `*.js` или `styles.css` сопровождается инкрементом
`?v=NN` на всех `<script>` и `<link>` в `index.html`. Это нужно, чтобы
WebView и браузеры подхватывали свежий файл, не залипая на старом.
